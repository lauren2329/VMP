Google Calendar Sync

Requires a Google Cloud OAuth Client ID and this page served over http/https (not opened directly as a file). See setup notes below.

Not connected.
One-time Google Cloud setup steps
  1. In Google Cloud Console, create/select a project and enable the "Google Calendar API".
  2. Configure the OAuth consent screen and add the calendar scope.
  3. Create an OAuth 2.0 Client ID of type "Web application".
  4. Under "Authorized JavaScript origins", add the exact URL this file is served from (e.g. http://localhost:8000) — file:// will not work.
  5. Paste the generated Client ID above, save, then click Connect.

Cross-Device Sync

This keeps everything in this calendar — goals, obligations, shifts, Mandala charts, all of it — updated live across every device and every person you give access to, using your own free Firebase database. This is separate from Google Calendar sync above (that one is just for calendar events).

Not connected.

Workspace

Everyone who shares a Workspace ID sees the same live data. If you're the first person setting this up, create a workspace and share its ID with your team. If someone shared an ID with you, paste it below and join instead.

People with Access

Owner and Editor can read and write everything, including managing this list. Viewer can see everything but every input, checkbox, and save action is disabled for them — enforced both in the interface and by the database itself, so it can't be bypassed from the browser console. There's still no owner-only lock on managing this list itself — any Owner or Editor can add, remove, or re-role anyone, including each other.

One-time Firebase setup steps
  1. Go to console.firebase.google.com, click "Add project," and either create a new one or import the same Google Cloud project you used for Calendar sync.
  2. In the left sidebar, go to Build → Firestore Database → Create database. Choose a location, start in "production mode."
  3. Go to the Firestore "Rules" tab and replace the contents with the rule shown below, then click Publish.
  4. Go to Build → Authentication → Get started → enable "Google" as a sign-in provider.
  5. Go to Project settings (gear icon) → General → scroll to "Your apps" → click the web icon (</>) to register a web app → copy the firebaseConfig object shown.
  6. Paste that object above, click Save Firebase Config, then Sign in with Google, then click "Create This Workspace."
  7. To add someone: give them this same firebaseConfig text, your Workspace ID (Copy button above), and add their email under "People with Access." They paste the config into their own copy of this file, sign in with their own Google account, and join your Workspace ID.

Firestore Rules to paste in step 3 — this gives every listed member read access, but only Owners and Editors can write; Viewers are blocked at the database level regardless of what the interface shows:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /calendarData/{workspaceId} {
      allow get: if request.auth != null && (request.auth.uid == workspaceId || (resource != null && request.auth.token.email in resource.data.members));
      allow create: if request.auth != null && request.auth.uid == workspaceId;
      allow update: if request.auth != null && resource.data.members[request.auth.token.email] in ['owner', 'editor'];
    }
  }
}
